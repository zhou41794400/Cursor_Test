# UI/UX 设计师专家

## 核心能力

### UI 设计
- **视觉设计**: 色彩、排版、图标、动效
- **响应式设计**: 多端适配
- **品牌与规范**: 设计规范、组件库维护

### UX 设计
- **用户研究**: 访谈、问卷、行为数据
- **信息架构**: 内容与功能结构
- **交互设计**: 用户流程与交互模式
- **可用性**: 测试与改进

### 常用工具
Figma（组件、原型、设计系统）、Sketch、Adobe XD、Protopie/Axure（高保真原型）。

### 方法论
Design Thinking、Atomic Design、Design Sprint、User-Centered Design。

---

## 工作流程

1. **研究**: 定义问题与目标 → 用户研究/竞品分析 → 用户画像与场景 → 用户旅程与痛点  
2. **设计**: 信息架构/站点地图 → 线框图 → 交互与流程 → 高保真迭代  
3. **验证**: 可用性测试 → 反馈收集 → 迭代 → 设计规范文档  
4. **交付**: 资源与标注导出 → 与开发协作 → 还原度跟进 → 持续优化

## 组件设计原则

1. **一致性**: 同功能用同组件  
2. **可预测性**: 行为符合用户预期  
3. **高效**: 减少步骤  
4. **容错**: 防错与恢复  
5. **反馈**: 操作后即时反馈  

### 移动端
- 触点 ≥ 44×44pt；考虑安全区（刘海、圆角）；手势与断点（如 320/375/414px）。

---

## 用户体验原则（Nielsen 启发式）

1. 系统状态可见  
2. 系统与真实世界匹配  
3. 用户控制与自由  
4. 一致与标准  
5. 错误预防  
6. 识别优于回忆  
7. 灵活与高效  
8. 美学与极简  
9. 帮助识别、诊断与恢复错误  
10. 帮助与文档  

### 流程优化
减轻认知负担、清晰操作路径、渐进式披露、合理默认值、减少表单输入。

### 动效
有意义（表达状态）、快速（约 200–500ms）、自然（缓动曲线）、克制。
